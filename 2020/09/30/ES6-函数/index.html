一、参数默认值

1.1 使用

  1. 在书写形参时，直接给形参赋值，附的值即为默认值

  2. 这样一来，当调用函数时，如果没有给对应的参数赋值（给它的值是undefined），则会自动使用默认值。

1.2 [扩展]对arguments的影响

  1. 只要给函数加上参数默认值，该函数会自动变量严格模式下的规则：arguments和形参脱离(以后arguments就不要用了，后面有更好的方法)

1.3 [扩展]留意暂时性死区

  1. 形参和ES6中的let或const声明一样，具有作用域，并且根据参数的声明顺序，存在暂时性死区。

二、剩余参数

2.1 数组

  1. 参数传递时，实参可以是数组，但是不方便

  2. 有个原则，写函数时一定要让调用的人爽，因为函数本身就是被调用的，调用的场景远远大于函数定义的场景，函数只定义一次，但是会调用多次

2.2 arguments的缺陷

  1. 如果和形参配合使用，容易导致混乱

  2. 从语义上，使用arguments获取参数，由于形参缺失，无法从函数定义上理解函数的真实意图

2.3 剩余参数

  1. ES6的剩余参数专门用于手机末尾的所有参数，将其放置到一个形参数组中。

  2. 语法: function (...形参名){}

  3. 细节

  • 一个函数，仅能出现一个剩余参数

  • 一个函数，如果有剩余参数，剩余参数必须是最后一个参数

三、展开运算符

3.1 剩余参数出现的问题

3.6 注意

  2. 用展开运算符实现克隆，可以在新对象中添加同名属性来覆盖克隆过来的属性值，而不破坏旧对象

五、箭头函数

5.1 回顾：this指向

  1. 通过对象调用函数，this指向对象(如果是DOM事件函数，this指向事件源)

  2. 直接调用函数，this指向全局对象

  3. 如果通过new调用函数，this指向新创建的对象

  4. 如果通过apply、call、bind调用函数，this指向指定的数据

5.2 以前避免this指向错误的方法

  1. 将this保存到一个变量内，如_this，然后在其他的地方用_this来避免this指向错误

  2. 缺点：如果需要保存的this过多，那么就会定义很多变量来保存this值

5.3 解决办法：箭头函数

  1. 箭头函数是一个函数表达式，理论上，任何使用函数表达式的场景都可以使用箭头函数(函数表达式是不会提升的)

  2. 完整语法：(参数1, 参数2, ...)=>{//函数体}

  • 如果参数只有一个，可以省略小括号:参数 => {}

  • 如果箭头函数只有一条返回语句，可以省略大括号，和return关键字:参数 => 返回值

  • 如果返回值是对象，系统会认为 参数=>{}是一个箭头函数，里面不能写键值对，所以报错

  • 解决办法：要在对象外面加上括号

5.6 注意细节

  1. 箭头函数函数体中this，取决于箭头函数定义的位置的this，与如何调用无关（将箭头函数换成this，这个this指向啥，箭头函数中的this就指向啥）
  • 真实情况：箭头函数中，不存在this、arguments、new.target，如果使用了，则使用的是函数外层的对应的this、arguments、new.target，箭头函数就像是临时用一下，用完就扔了

  1. 箭头函数没有原型

  2. 箭头函数不能作用构造函数使用

  3. 对象属性不会用箭头函数

5.7 应用场景

  1. 临时性使用的函数，并不会可以调用它，比如：

  • 事件处理函数

  • 异步处理函数，例：定时器

  • 其他临时性的函数

  1. 为了绑定外层this的函数

  2. 在不影响其他代码的情况下，保持代码的简洁，最常见的，数组方法中的回调函数
