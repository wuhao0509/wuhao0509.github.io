一、HTML5页面

1.1 写出HTML5页面标准结构？

1.2 DOCTYPE作用？

  1. <!DOCTYPE>必须放在第一行，这并不是一个HTML标签
  2. <!DOCTYPE>叫做文件类型定义(DTD)，作用是告诉浏览器该文件类型，让浏览器解析器 知道用哪个规范来解析文档

1.3 标准模式与混杂模式如何区分？他们有什么意义？

  1. 标准模式和混杂模式
  • 标准模式：浏览器按照W3C标准解析代码
  • 混杂模式：浏览器用自己的方式解析代码
  1. 区分标准模式和混杂模式
  • 文档包含严格的DOCTYPE，那么它一般以标准模式呈现
  • DTD不存在或者格式不正确——混杂模式
  • 有URI的过渡DTD——标准模式
  • 没有URI的过渡DTD——混杂模式
  1. HTML5没有严格和混杂之分
  • HTML5没有DTD，因此也就没有严格模式与混杂模式的区别
  • HTML5有相对宽松的语法，已经尽可能大的实现了向后兼容
  1. 意义： 严格模式与混杂模式存在的意义与其来源 密切相关，如果说只存在严格模式，那么许多旧网站必然受到影响，如果只存在混杂模式，那么会回到当时浏览器大战时的混乱，每个浏览器都有自己的解析模式。

二、语义化标签

2.1 标签语义化是指什么？

  1. 就是让标签有含义，给某块内容用上一个最恰当最合适 的标签，使页面有良好的结构
  2. 页面元素有含义，不论是谁都能够看懂这块内容是什么，有利于搜索引擎优化

2.2 标签语义化有什么好处

  1. HTML结构清晰
  2. 代码可读性较好
  3. 无障碍阅读
  4. 搜索引擎可以根据标签的语言确定上下文和权重问题
  5. 移动设备能够更完美的展现网页（对css支持较弱的设备）
  6. 便于团队维护和开发

三、属性

3.1 元素alt与title异同？

  1. alt作为图片的替代文字出现，title作为图片的解释文字出现。
  2. alt属性应用较少，如img、area、input中；title应用较多，如a、form、input、还有div、p这些块级元素都可以用。
  3. alt仅作为标签属性，title也可以是标签，它是网页的标题标签。

五、canvas与svg

5.1 什么是canvas

  1. Canvas是h5新增的一个标签
  2. HTML5的canvas元素使用JS在网页上绘制图像。
  3. 画布是一个矩形区域，您可以控制其每一像素。
  4. canvas拥有多种绘制路径、矩形、圆形、字符以及添加图像的方法。

5.2 什么是svg？

  1. SVG指可伸缩矢量图形，用于定义用于网络的基于矢量的图形
  2. 格式：XML
  3. SVG在放大 或改变尺寸 的情况下，其图形质量 不会有损失
  4. SVG是万维网联盟的标准

5.3 与其他图像格式相比(JPEG、GIF)有哪些优势？

  1. SVG可伸缩，可在图像质量不下降的情况下被放大
  2. 并且在任何的分辨率下被高质量地打印
  3. SVG可通过文本编辑器来创建和修改
  4. SVG可被搜索、索引、脚本化或压缩

5.4 canvas与svg的区别？

canvas

  1. 依赖分辨率
  2. 不支持事件处理器
  3. 弱的文本渲染能力
  4. 能够以 .png 或 .jpg 格式保存结果图像
  5. 最适合图像密集型的游戏，其中的许多对象会被频繁重绘

svg

  1. 不依赖分辨率
  2. 支持事件处理器
  3. 最适合带有大型渲染区域的应用程序（比如谷歌地图）
  4. 复杂度高会减慢渲染速度（任何过度使用DOM的应用都不快）
  5. 不适合游戏应用

5.5 在CSS3中可以通过linear-gradient创建线性渐变，canvas可以通过什么创建线性渐变对象？并通过什么方法添加颜色？

  1. createLinearGradient()、addColorStop()
