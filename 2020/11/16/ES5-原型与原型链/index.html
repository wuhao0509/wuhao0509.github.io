原型习题

习题1： 控制台输入person.name，应该输出什么

答案：
“cherry”

解析：
控制台输入的内容相当于是在程序逻辑最后面的代码块，所以”cherry”会覆盖”sunny”

习题2： 控制台输入person.name，person.name应该输出什么

答案：
“cherry”

解析：
控制台输入的内容相当于是在程序逻辑最后面的代码块，所以”cherry”会覆盖”sunny”

习题3：

答案：
“sunny”

解析：
一开始，函数原型的name是”sunny”，用构造函数构造对象后，对象person继承了”sunny”，然后函数原型指向了一个新的对象（因为函数原型是一个对象，是引用值），此时person已经继承了原先的对象，即使函数原型的指向发生了变化，但是也不能修改了

原型链的增删改查

  1. 调用的修改： 用子孙修改原型的属性时，原型的属性只能用引用值，这里father的fortune属性是对象，即引用值，如果fortune = 100；则不能修改

  2. 覆盖性的修改： son.fortune = 100; 是给son添加了一个fortune属性，father的fortune属性没有被修改

创建对象的第四种方法

  1. Object.create(原型)

    原型必须是一个Object对象，或null，用null创建的对象没有原型，可以给对象人为创建一个__proto__,但是不具有继承特性

  2. 绝大多数对象最终都会继承自Object.prototype，这句话对吗

    错 用创建对象的第四种方法var obj = Object.create(null);创建的对象没有原型，可以给obj人为创建一个__proto__,但是不具有继承特性

原型链终端

  1. var obj = {},var obj = new Object();的原型？

    都是Object.prototype var obj = {}与var obj = new Object();效果是一样的，当用字面量形式创建一个对象时，系统会将它转换成var obj = new Object();，所以极少用系统构造函数创建对象

  2. 原型链终端的推导

    以构造函数function Person(){}为例，自己写的构造函数默认原型为空对象，即Person.prototype = {};,而空对象实际上是由系统构造函数new Object();构造出来的，系统构造函数的原型为Object.prototype;,所以原型链终端是Object.prototype;

toString()拓展

  1. 123.toString();能输出吗？

    不能 由于数学运算中点”.”的优先级是最高的，所以浏览器将123.toString();默认为浮点型，所以浏览器会报错，所以数字需要转换成变量浏览器才能识别，如var num = 123; num.toString()

  2. var obj = {}; obj.toString();输出结果？

    [object Object] 这个toString()方法毋庸置疑的来源于原型链的终端

  3. var num = 123; num.toString()输出结果?

    输出字符串”123”

  4. 由问题2可以推出问题3应该输出[object Number]，为什么问题2结果是”123”？
