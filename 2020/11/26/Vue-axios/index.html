十、错误处理

在请求错误时进行的处理

  1. request / response 是error的上下文，标志着请求发送 / 得到响应

  2. 在错误中，如果响应有值，则说明是响应时出现了错误。

      如果响应没值，则说明是请求时出现了错误。

  3. 在错误中，如果请求无值，则说明是请求未发送出去，如取消请求。

  1. 在实际开发过程中，一般在拦截器中统一添加错误处理

  2. 请求拦截器中的错误，会当请求未成功发出时执行，但是要注意的是：取消请求后，请求拦截器的错误函数也不会执行，因为取消请求不会抛出异常，axios对其进行了单独的处理。

  3. 在更多的情况下，我们会在响应拦截器中处理错误。

十一、axios 预检

  1. 当axios的请求为非简单请求时，浏览器会进行预检，及发送OPTIONS请求。请求到服务器，询问是否允许跨域。如果响应中允许预检中请求的跨域行为，则浏览器会进行真正的请求。否则会报405错误。
